# Contributing to BuildPilot PoE

Thanks for your interest in contributing.

We want this project to stay practical, lightweight, and useful for Path of Exile players who follow builds during gameplay.

## Ground rules

Please keep contributions aligned with the product direction:

- overlay assistance, not automation
- clear UX over feature bloat
- lightweight and maintainable code
- player-controlled interactions only

This project should help users follow a build more easily.
It should not automate gameplay.

## Ways to contribute

You can help by contributing in areas like:

- bug reports
- feature suggestions
- UI and UX improvements
- documentation
- PoB parsing improvements
- overlay performance
- testing on different Windows setups

## Before opening an issue

Please check whether the issue already exists.

When opening a bug report, include:

- what happened
- what you expected to happen
- steps to reproduce
- app version
- Windows version
- screenshots or logs if possible

## Before opening a pull request

Please open an issue first for:

- new features
- architectural changes
- large refactors

Small fixes and documentation improvements can usually be opened directly.

## Pull request guidelines

Please keep pull requests:

- focused
- small when possible
- clearly described
- easy to review

Include:

- summary of the change
- reason for the change
- screenshots for UI updates
- testing notes when relevant

## Code style

General expectations:

- prefer readable code over clever code
- keep components small and composable
- avoid unnecessary dependencies
- document non-obvious decisions
- keep naming consistent

## Product boundaries

The following are out of scope unless explicitly discussed and approved:

- gameplay automation
- bot-like features
- input simulation for game actions
- anything that could push the tool away from passive assistance

## Development setup

Project setup instructions will be added as the codebase is structured.

For now, contributors can help most by discussing:

- parser design
- overlay UX
- progression modeling
- desktop architecture

## Community standards

Please be respectful and constructive.

Good feedback is specific, actionable, and kind.
